# WPR 웹앱 — 담당자별 R&R 뷰 추가 개발 지시서

## 🎯 목적

기존 **기능별 분류** 탭(공문번호, 명함 신청 등 업무 아이콘 그리드)에  
**담당자별 분류** 탭을 나란히 추가한다.  

담당자 탭에서는 4명의 담당자 카드를 보여주고,  
각 담당자 클릭 → 해당 담당자의 R&R 목록 → 항목 클릭 시  
**기존 기능별 탭의 상세 매뉴얼과 동일한 콘텐츠**가 호출된다.  
즉, 콘텐츠는 공유하고 진입 경로만 달라지는 구조다.

---

## 🚫 절대 건드리지 말 것

- 기존 일러스트 배경 (작업 중인 직원 이미지, 책상 씬)
- 상단 네비게이션 바 (Cushman & Wakefield 로고, WPR / FAQ 메뉴)
- macOS 윈도우 프레임 (빨강/노랑/초록 버튼, "WPR 업무 가이드" 타이틀바)
- 기존 기능별 아이콘 그리드 레이아웃 및 각 기능의 상세 패널 / 이메일 템플릿
- 기존 `tasks.js`의 데이터 구조 (읽기만 할 것, 수정 금지)

---

## 🛠️ 기술 스택

- **Framework**: React 18 + Vite (기존 프로젝트 그대로 사용)
- **Styling**: Tailwind CSS CDN + 기존 인라인 CSS 패턴 유지
- **상태 관리**: React useState (기존 방식 유지, 전역 상태 라이브러리 추가 금지)
- **신규 파일만 추가**: 기존 파일은 최소한으로만 수정

### Phase 2 — `WindowTabBar` 컴포넌트 (탭 전환 UI)

**목표**: macOS 윈도우 상단 타이틀바 바로 아래에  
"기능별 분류" / "담당자별 분류" 전환 탭이 추가된다.

**`WindowTabBar.jsx` 전체 코드**:
```jsx
// src/components/WindowTabBar.jsx
export default function WindowTabBar({ viewMode, onChangeView }) {
  const tabs = [
    { id: 'function', label: '🗂️ 기능별 분류' },
    { id: 'person',   label: '👤 담당자별 분류' },
  ];

  return (
    <div style={{
      display: 'flex',
      borderBottom: '1px solid rgba(0,0,0,0.1)',
      backgroundColor: '#F5F5F5',  // macOS 윈도우 내부 배경색에 맞춤
      padding: '0 16px',
    }}>
      {tabs.map(tab => (
        <button
          key={tab.id}
          onClick={() => onChangeView(tab.id)}
          style={{
            padding: '8px 18px',
            fontSize: '13px',
            fontWeight: viewMode === tab.id ? '600' : '400',
            color: viewMode === tab.id ? '#1A1A2E' : '#888888',
            background: 'transparent',
            border: 'none',
            borderBottom: viewMode === tab.id
              ? '2px solid #1A1A2E'
              : '2px solid transparent',
            cursor: 'pointer',
            transition: 'all 0.15s ease',
            letterSpacing: '-0.2px',
          }}
        >
          {tab.label}
        </button>
      ))}
    </div>
  );
}
```

**`App.jsx` 수정 사항**:
1. `import WindowTabBar from './components/WindowTabBar'` 추가
2. `useState`에 `viewMode` 상태 추가: `const [viewMode, setViewMode] = useState('function')`
3. macOS 윈도우 타이틀바(`WPR 업무 가이드` 헤더) 바로 아래에 `<WindowTabBar>` 삽입:
```jsx
{/* 타이틀바 이후, 콘텐츠 영역 이전에 삽입 */}
<WindowTabBar
  viewMode={viewMode}
  onChangeView={(mode) => {
    setViewMode(mode);
    setSelectedPerson(null);   // 탭 전환 시 담당자 선택 초기화
    setSelectedTask(null);     // 탭 전환 시 기능 선택 초기화
  }}
/>
```

**완료 확인**: 윈도우 상단에 두 개 탭이 표시되고, 클릭 시 active 탭이 시각적으로 구분된다.

### Phase 3 — `PersonGrid` 컴포넌트 (담당자 카드 그리드)

**목표**: 담당자별 탭 선택 시 4명의 담당자 카드가 기존 아이콘 그리드 자리에 표시된다.

**레이아웃**: 2열 × 2행 그리드, 기존 아이콘 그리드와 동일한 컨테이너 크기 사용

**`PersonGrid.jsx` 전체 코드**:
```jsx
// src/components/PersonGrid.jsx
import { persons } from '../data/persons';

export default function PersonGrid({ onSelectPerson }) {
  return (
    <div style={{
      display: 'grid',
      gridTemplateColumns: '1fr 1fr',
      gap: '12px',
      padding: '16px',
      overflowY: 'auto',
    }}>
      {persons.map(person => (
        <PersonCard
          key={person.id}
          person={person}
          onClick={() => onSelectPerson(person)}
        />
      ))}
    </div>
  );
}

function PersonCard({ person, onClick }) {
  return (
    <button
      onClick={onClick}
      style={{
        display: 'flex',
        flexDirection: 'column',
        alignItems: 'flex-start',
        gap: '8px',
        padding: '16px',
        background: 'white',
        border: `1.5px solid ${person.color}33`,   // color + 20% opacity
        borderRadius: '12px',
        cursor: 'pointer',
        transition: 'all 0.18s ease',
        textAlign: 'left',
        boxShadow: '0 1px 4px rgba(0,0,0,0.06)',
      }}
      onMouseEnter={e => {
        e.currentTarget.style.borderColor = person.color;
        e.currentTarget.style.boxShadow = `0 4px 16px ${person.color}22`;
        e.currentTarget.style.transform = 'translateY(-2px)';

        e.currentTarget.style.boxShadow = '0 1px 4px rgba(0,0,0,0.06)';
        e.currentTarget.style.transform = 'translateY(0)';
      }}
    >
      {/* 아바타 + 이름 행 */}
      <div style={{ display: 'flex', alignItems: 'center', gap: '10px' }}>
        <div style={{
          width: '40px', height: '40px',
          borderRadius: '50%',
          background: `${person.color}18`,
          border: `2px solid ${person.color}`,
          display: 'flex', alignItems: 'center', justifyContent: 'center',
          fontSize: '16px', fontWeight: '700', color: person.color,

1. `import PersonGrid from './components/PersonGrid'` 추가

3. 윈도우 내부 콘텐츠 영역에서 viewMode에 따른 조건부 렌더링 추가:

{/* 기존 아이콘 그리드가 렌더링되던 자리를 아래로 교체 */}
{viewMode === 'function' ? (

### Phase 4 — `PersonDetailPanel` 컴포넌트 (R&R 목록)

**목표**: 담당자 카드 클릭 시 해당 담당자의 R&R 목록이 표시된다.  
R&R 항목 클릭 시 **기존 기능별 탭과 동일한 상세 패널(RightSlidePanel)**이 열린다.

**핵심 로직**: `taskId`로 `tasks.js`에서 해당 task 객체를 조회하여 `setSelectedTask`에 전달.

          letterSpacing: '0.8px', padding: '4px 4px 8px',
        }}>
          담당 업무 ({person.responsibilities.length})
        </div>

        {person.responsibilities.map((resp, index) => {
          const task = getTaskById(resp.taskId);
          const isLinked = !!task;  // task가 존재하면 연결 가능

          return (
            <RRItem
              key={resp.taskId}
              resp={resp}
              task={task}
              isLinked={isLinked}
              personColor={person.color}
              index={index}
              onSelectTask={onSelectTask}
            />
          );
        })}
      </div>
    </div>
  );
}

function RRItem({ resp, task, isLinked, personColor, index, onSelectTask }) {
  const handleClick = () => {
    if (!isLinked) return;
    onSelectTask(task);   // 기존 RightSlidePanel을 task로 열기
  };

  return (
    <button
      onClick={handleClick}
      disabled={!isLinked}
      style={{
        display: 'flex',
        alignItems: 'center',
        gap: '12px',
        padding: '12px 14px',
        background: 'white',
        border: `1px solid ${isLinked ? 'rgba(0,0,0,0.08)' : 'rgba(0,0,0,0.04)'}`,
        borderRadius: '10px',
        cursor: isLinked ? 'pointer' : 'default',
        textAlign: 'left',
        transition: 'all 0.15s ease',
        opacity: isLinked ? 1 : 0.5,
        width: '100%',
      }}
      onMouseEnter={e => {
        if (!isLinked) return;
        e.currentTarget.style.borderColor = personColor;
        e.currentTarget.style.background = `${personColor}06`;
        e.currentTarget.style.transform = 'translateX(3px)';
      }}
      onMouseLeave={e => {
        e.currentTarget.style.borderColor = 'rgba(0,0,0,0.08)';
        e.currentTarget.style.background = 'white';
        e.currentTarget.style.transform = 'translateX(0)';
      }}
    >
      {/* 순서 번호 */}
      <div style={{
        width: '24px', height: '24px',
        borderRadius: '50%',
        background: `${personColor}16`,
        border: `1.5px solid ${personColor}40`,
        display: 'flex', alignItems: 'center', justifyContent: 'center',
        fontSize: '11px', fontWeight: '700', color: personColor,
        flexShrink: 0,
      }}>
        {index + 1}
      </div>

      {/* 텍스트 */}
      <div style={{ flex: 1 }}>
        <div style={{
          fontSize: '13px', fontWeight: '600', color: '#1A1A2E',
          marginBottom: '2px',
        }}>
          {resp.label}
        </div>
        <div style={{ fontSize: '11px', color: '#888' }}>
          {resp.description}
        </div>
      </div>

      {/* 연결 상태 아이콘 */}
      <div style={{ fontSize: '14px', color: isLinked ? personColor : '#CCC', flexShrink: 0 }}>
        {isLinked ? '→' : '—'}
      </div>
    </button>
  );
}
```

**완료 확인**:  
- 담당자 카드 클릭 → R&R 목록이 헤더(뒤로가기 + 이름)와 함께 표시된다  
- R&R 항목 클릭 → 기존 기능별 탭과 동일한 `RightSlidePanel`이 열린다  
- 뒤로가기 클릭 → 담당자 카드 그리드로 돌아간다

### Phase 5 — `App.jsx` 통합 및 상태 흐름 완성

**목표**: 모든 컴포넌트가 연결되어 두 탭이 완전히 작동한다.

**`App.jsx` 최종 구조** (기존 코드 기반으로 아래 변경사항만 적용):

```jsx
// 추가할 import
import { useState } from 'react';
import WindowTabBar from './components/WindowTabBar';
import PersonGrid from './components/PersonGrid';
import PersonDetailPanel from './components/PersonDetailPanel';
import { persons } from './data/persons';   // 사용 여부에 따라 포함

// 추가할 상태 (기존 selectedTask, selectedStep 옆에)
const [viewMode, setViewMode] = useState('function');
const [selectedPerson, setSelectedPerson] = useState(null);

// 탭 전환 핸들러
const handleViewChange = (mode) => {
  setViewMode(mode);
  setSelectedPerson(null);
  setSelectedTask(null);    // 기존 상태 이름으로 교체
};

// 담당자 R&R → task 열기 핸들러
const handlePersonTaskSelect = (task) => {
  // viewMode를 'function'으로 바꾸지 않음 — 담당자 탭에서 그대로 패널 열기
  setSelectedTask(task);    // 기존 RightSlidePanel이 이미 selectedTask를 바라보고 있음
};
```

**JSX 구조 (macOS 윈도우 내부)**:
```jsx
<div className="macos-window">
  {/* 타이틀바 — 건드리지 않음 */}
  <TitleBar />

  {/* ✅ 신규: 탭 바 삽입 */}
  <WindowTabBar viewMode={viewMode} onChangeView={handleViewChange} />

  {/* 콘텐츠 영역 */}
  <div className="window-content">
    {viewMode === 'function' ? (
      // ✅ 기존 기능별 그리드 — 건드리지 않음
      <ExistingFunctionGrid
        onSelectTask={setSelectedTask}
      />
    ) : (
      // ✅ 신규: 담당자별 뷰
      selectedPerson ? (
        <PersonDetailPanel
          person={selectedPerson}
          onBack={() => setSelectedPerson(null)}
          onSelectTask={handlePersonTaskSelect}
        />
      ) : (
        <PersonGrid onSelectPerson={setSelectedPerson} />
      )
    )}
  </div>

  {/* RightSlidePanel — viewMode 무관하게 selectedTask가 있으면 항상 표시 */}
  {selectedTask && (
    <RightSlidePanel
      task={selectedTask}
      onClose={() => setSelectedTask(null)}
      onSelectStep={setSelectedStep}
    />
  )}
</div>

{/* StepDetailModal — 기존 그대로 */}
{selectedStep && (
  <StepDetailModal
    step={selectedStep}
    task={selectedTask}
    onClose={() => setSelectedStep(null)}
  />
)}
```

## ⚠️ 주의사항 및 예상 오류

| 문제 | 원인 | 해결법 |
|---|---|---|
| R&R 항목 클릭해도 패널 안 열림 | `taskId`가 `tasks.js`의 실제 `id`와 불일치 | `persons.js`의 `taskId`를 `tasks.js` 실제 `id`로 수정 |
| 담당자 탭에서 패널 열 때 기능별 탭으로 전환됨 | `handlePersonTaskSelect`에서 `setViewMode('function')` 호출 | viewMode 변경 없이 `setSelectedTask(task)`만 호출 |
| PersonGrid가 기존 그리드와 레이아웃 충돌 | 컨테이너 높이 고정값 | `height: 100%` + `overflow-y: auto` 확인 |
| 탭 전환 시 이전 선택 상태 잔류 | `handleViewChange`에서 초기화 누락 | `setSelectedTask(null)`, `setSelectedPerson(null)` 동시 호출 |
| 모바일에서 PersonCard 글자 잘림 | 카드 너비 부족 | 모바일에서 1열로 전환 + 카드 `min-height` 설정 |
